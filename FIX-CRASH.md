# Исправление ошибки "app crashed"

Если сервер падает с ошибкой `[nodemon] app crashed`, выполните следующие шаги:

## 🔧 Быстрое исправление

### Вариант 1: Автоматический скрипт (рекомендуется)

1. **Остановите все процессы Node.js:**
   - Нажмите `Ctrl+C` в терминале, где запущен сервер
   - Или закройте все окна терминала

2. **Запустите скрипт исправления:**
   ```bash
   fix-server.bat
   ```
   Или просто дважды кликните на файл `fix-server.bat`

3. **Запустите сервер заново:**
   ```bash
   npm run dev
   ```

---

### Вариант 2: Ручное исправление

1. **Остановите все процессы Node.js:**
   ```bash
   # В PowerShell:
   taskkill /F /IM node.exe
   ```

2. **Удалите старую базу данных:**
   - Удалите файл `server/database/database.sqlite`
   - Или запустите: `reset-db.bat`

3. **Протестируйте базу данных:**
   ```bash
   node test-db.js
   ```
   Должно вывести: `✅ Тест завершен успешно!`

4. **Запустите сервер:**
   ```bash
   npm run dev
   ```

---

## 🔍 Диагностика

Если проблема сохраняется, проверьте:

1. **Порт 5000 занят:**
   ```bash
   netstat -ano | findstr :5000
   ```
   Если порт занят, завершите процесс или измените порт в `.env`:
   ```
   PORT=5001
   ```

2. **База данных повреждена:**
   - Удалите `server/database/database.sqlite`
   - Запустите `node test-db.js` для пересоздания

3. **Проверьте логи:**
   - Запустите `node server/index.js` (без nodemon)
   - Посмотрите на точное сообщение об ошибке

---

```
✅ Все таблицы созданы

```

---

## 📞 Если ничего не помогает

1. Убедитесь, что установлены все зависимости:
   ```bash
   npm install
   ```

2. Проверьте версию Node.js (должна быть >= 14):
   ```bash
   node --version
   ```

3. Проверьте, что все файлы на месте:
   - `server/index.js`
   - `server/database/init.js`
   - `server/routes/*.js`
