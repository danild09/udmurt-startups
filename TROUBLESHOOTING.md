# Решение проблем с запуском сервера

## Проблема: Сервер не запускается

### 1. Проверьте установку зависимостей

```bash
npm install
```

Если есть ошибки, попробуйте:
```bash
npm cache clean --force
npm install
```

### 2. Проверьте, не занят ли порт 5000

**Windows:**
```bash
netstat -ano | findstr :5000
```

Если порт занят, либо остановите процесс, либо измените порт в файле `.env`:
```
PORT=3001
```

### 3. Проверьте наличие Node.js

```bash
node --version
npm --version
```

Должна быть версия Node.js 14 или выше.

### 4. Ошибки с базой данных

Если возникают ошибки с SQLite:
- Убедитесь, что папка `server/database` существует
- Проверьте права на запись в эту папку
- Попробуйте удалить `server/database/database.sqlite` и запустить снова

### 5. Ошибки с модулями

Если видите ошибки типа "Cannot find module":
```bash
# Удалите node_modules и установите заново
rm -rf node_modules
npm install
```

**Windows:**
```cmd
rmdir /s /q node_modules
npm install
```

### 6. Запуск через скрипт проверки

```bash
node check-server.js
```

Этот скрипт проверит все зависимости и файлы.

### 7. Запуск через batch файл (Windows)

Дважды кликните на `start-server.bat` или запустите в командной строке.

### 8. Типичные ошибки

**"EADDRINUSE"** - порт занят
- Решение: измените PORT в .env или остановите процесс на порту 5000

**"MODULE_NOT_FOUND"** - модуль не найден
- Решение: запустите `npm install`

**"EACCES"** - нет прав доступа
- Решение: запустите от имени администратора или измените порт

**Ошибки с путями (кириллица в пути)**
- Решение: переместите проект в папку с английским названием

### 9. Альтернативный запуск

Если стандартный запуск не работает, попробуйте:

```bash
# Прямой запуск Node.js
node server/index.js

# Или с явным указанием порта
PORT=3001 node server/index.js
```

### 10. Проверка логов

Сервер выводит сообщения в консоль. Обратите внимание на:
- ✅ Сервер запущен на порту...
- ✅ Подключение к базе данных установлено
- ✅ База данных инициализирована

Если видите ошибки, скопируйте их и проверьте в этом руководстве.

## Получение помощи

Если проблема не решена:
1. Запустите `node check-server.js` и скопируйте вывод
2. Запустите сервер и скопируйте все сообщения об ошибках
3. Проверьте версию Node.js: `node --version`
4. Проверьте версию npm: `npm --version`
